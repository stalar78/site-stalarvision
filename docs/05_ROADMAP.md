5. подготовить вторую страницу услуги — технического аудита сайта;

7. настроить расширенные цели Яндекс Метрики для формы и контактных CTA;
8. отдельно разобрать вывод `npm audit` без автоматического `--force`-обновления.

## Подтверждённая SEO-рамка

На текущем этапе согласовано:
- первичная география — Санкт-Петербург и Ленинградская область;
- работа выполняется удалённо;
- приоритет услуг:
  1. доработка существующих сайтов;
  2. технический аудит сайта;
  3. разработка нового сайта;
  4. веб-приложения и личные кабинеты;
  5. интеграции и автоматизация;
- предпочтительная загрузка — один крупный или два средних проекта;
- все реальные собственные и выполненные проекты могут рассматриваться как кандидаты для кейсов, но каждый кейс публикуется только после проверки фактов и формулировок.

## Открытые вопросы

Пока остаются открытыми:
- нужен ли пререндеринг в текущем Vite-проекте или позже станет оправдан переход публичного слоя на другую архитектуру;
- какие реальные проекты первыми оформить как полноценные кейсы;
- оставить ли mixed RU/EN framing в Portfolio (`Representative examples`) или перевести его полностью на русский;
- нужен ли дополнительный legal/privacy слой сверх уже реализованного;
- какие именно зависимости формируют текущие предупреждения `npm audit` и применимы ли они к production-сценарию сайта.

## Что не является приоритетом прямо сейчас

Пока не приоритет:
- большой редизайн;
- тяжёлый backend для сайта;
- сложная CMS;
- большая админка;
- массовое создание десятков SEO-страниц без проверки первой страницы;
- миграция на Next.js только “на всякий случай”;
- `npm audit fix --force` без отдельного анализа breaking changes.

## Принцип следующего шага

Проект развивается по циклу:
1. одна обоснованная SEO-задача;
2. реализация кода через Codex;
3. build и push владельцем;
4. ревью актуального GitHub-состояния через GPT;
5. обновление docs через GPT непосредственно в GitHub;
6. безопасный production-деплой;
7. ручная проверка результата перед следующим этапом.
