# Product Requirements

## Основная цель сайта

Сайт должен:
- выглядеть как сильный персональный сайт разработчика,
- вызывать доверие,
- объяснять формат работы,
- помогать потенциальному клиенту быстро понять, с какими задачами сюда можно приходить,
- приводить обращения через email / форму / прямой контакт.

## Что сайт должен транслировать

Сайт должен передавать следующие смыслы:
- это сайт одного специалиста, а не команды;
- работа идёт напрямую;
- задачи можно обсуждать спокойно и по делу;
- можно прийти как с новым проектом, так и с доработкой существующего или запросом на аудит / технический разбор;
- важны ясность, структура, адекватный первый этап и поддерживаемый результат.

## Тональность

Тон сайта:
- спокойный,
- уверенный,
- современный,
- деловой,
- без агентского пафоса,
- без премиального пустого маркетинга,
- без шаблонных обещаний.

Предпочтительна подача от первого лица там, где это делает сайт живее и естественнее.

## Чего сайт не должен делать

Сайт не должен:
- прикидываться большим агентством;
- продавать через фальшивое социальное доказательство;
- содержать выдуманные обещания;
- содержать агрессивный маркетинговый тон;
- выглядеть как generic UI-шаблон без личности.

## Контентные ограничения

Любые неизвестные данные должны:
- либо оставаться нейтральными,
- либо откладываться до подтверждения,
- либо явно обозначаться как временные/иллюстративные, если речь о примерах.

## UX-требования

Сайт должен быть:
- аккуратным на мобильных устройствах,
- визуально чистым,
- понятным по структуре,
- без ломающихся карточек и переполнений,
- с рабочими CTA,
- с понятным контактным сценарием.

## Текущий продуктовый статус

На текущем этапе уже реализовано:
- опубликованная версия сайта,
- рабочий production-домен с HTTPS,
- privacy/legal page,
- 404,
- реальные контакты: email, Telegram, GitHub и телефон,
- рабочая форма через Web3Forms,
- честная секция с примерами формата проектов вместо фейковых кейсов,
- улучшенная мобильная адаптация,
- post-launch SEO/share polish: canonical, Open Graph meta, Twitter meta, `robots.txt`, `sitemap.xml`,
- privacy/legal тексты, согласованные с реальной работой формы,
- уточнённые верхнеуровневые CTA и формулировки под сценарии: новый проект / доработка / аудит / технический разбор,
- services, contact flow, representative examples и FAQ дополнительно отполированы под эту рамку.

Следующие шаги должны быть точечными и продуктово обоснованными.
